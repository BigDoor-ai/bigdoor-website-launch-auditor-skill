# Severity Rubric

## Critical

Must fix before launch.

Examples: public secret exposure, unprotected admin route, payment/security issue, broken checkout/signup/lead form, website cannot build, user data exposure, missing authorization on private data, severe legal risk for regulated claims.

## High

Should fix before launch.

Examples: poor mobile UX on main conversion flow, missing validation on important forms, missing SEO metadata on major pages, no privacy policy while collecting personal data, missing error handling for important flows.

## Medium

Should fix soon.

Examples: inconsistent UI components, missing empty/loading states, non-critical accessibility issues, performance improvement opportunities, minor documentation gaps.

## Low

Improvement opportunity.

Examples: nice-to-have copy improvement, additional analytics event, better internal naming, more helpful README section.

## Unverified

Cannot confirm from available code, config, command output, live URL, or documentation.
